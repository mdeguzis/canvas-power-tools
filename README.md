# Canvas Power Tools

A cross-browser extension (Chrome and Firefox) that adds persistent activity notifications and a settings panel to Canvas LMS.

## Features

- Standing notification badge when new Canvas activity is detected (announcements, grade releases, messages, file uploads)
- Configurable poll interval
- Dismiss button to clear the badge
- Settings popup with toggle and domain override
- Built on the native Canvas activity stream API - no scraping, no admin access required

## Project Structure

```
src/
  manifest.json       Extension manifest (Manifest V3)
  background.js       Service worker - manages the polling alarm
  content.js          Injected into Canvas pages - fetches activity and renders the badge
  popup/
    popup.html        Settings popup UI
    popup.css         Popup styles
    popup.js          Reads/writes chrome.storage.sync
  icons/              Extension icons (16, 48, 128px)
scripts/
  build.sh            Builds browser-specific zip files for distribution
dist/                 Generated by build.sh - not committed to git
```

## Development Setup

### Prerequisites

- Node is not required - this extension is plain JavaScript (no build step for development)
- `zip` (for the release build script)
- Python 3 (used by build.sh to patch the Firefox manifest)

### Load in Chrome (Developer Mode)

1. Open Chrome and go to `chrome://extensions`
2. Enable **Developer mode** (top right toggle)
3. Click **Load unpacked**
4. Select the `src/` folder in this repo
5. The extension icon appears in the toolbar

To reload after making changes: click the refresh icon on the extension card, then hard-reload the Canvas tab.

### Load in Firefox (Temporary Add-on)

1. Open Firefox and go to `about:debugging#/runtime/this-firefox`
2. Click **Load Temporary Add-on...**
3. Select `src/manifest.json`
4. The extension loads for the current Firefox session

Note: Temporary add-ons are removed when Firefox restarts. For persistent dev installs, use Firefox Developer Edition or sign the extension (see Publishing section below).

## Building Release Zips

Run the build script to generate distribution-ready zips for both browsers:

```bash
./scripts/build.sh
```

This produces:

```
dist/
  canvas-power-tools-<version>-chrome.zip
  canvas-power-tools-<version>-firefox.zip
```

The Firefox zip includes a `browser_specific_settings.gecko` block in the manifest, which Chrome ignores. Everything else is identical.

## Publishing

### Chrome Web Store

1. Go to the [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole)
2. Sign in with a Google account and pay the one-time $5 developer registration fee
3. Click **New Item** and upload `dist/canvas-power-tools-<version>-chrome.zip`
4. Fill in the store listing: description, screenshots (at least 1280x800 or 640x400), and category (Productivity)
5. Under **Privacy**, declare that the extension accesses user activity on Canvas sites
6. Submit for review - Chrome reviews typically take 1-3 business days for new submissions

For updates: bump the version in `src/manifest.json`, rebuild, and upload the new zip to the same listing.

### Firefox Add-ons (addons.mozilla.org)

Firefox requires extensions to be signed before they can be installed permanently (outside of Developer Edition).

1. Create an account at [addons.mozilla.org](https://addons.mozilla.org/developers/)
2. Click **Submit a New Add-on** and choose **On this site** for listed distribution
3. Upload `dist/canvas-power-tools-<version>-firefox.zip`
4. Fill in the listing: name, summary, description, and at least one screenshot
5. Add the source code zip if requested (Mozilla reviews source for some submissions)
6. Submit - initial reviews can take anywhere from a few hours to a couple of weeks

For updates: bump `src/manifest.json` version, rebuild, submit a new version through the developer hub.

#### Self-distributing on Firefox (skip the store)

If you just want to share it with a few people without going through AMO review:

1. Sign up at [addons.mozilla.org](https://addons.mozilla.org/developers/) and generate API credentials
2. Install [web-ext](https://extensionworkshop.com/documentation/develop/getting-started-with-web-ext/): `npm install -g web-ext`
3. Sign the zip: `web-ext sign --source-dir src/ --api-key <key> --api-secret <secret>`
4. This produces a signed `.xpi` file users can install by dragging it into Firefox

## How It Works

The extension uses the Canvas native REST API endpoint:

```
GET /api/v1/users/self/activity_stream
```

This endpoint is available to any logged-in Canvas user without special permissions. The fetch uses the user's active browser session cookie automatically since it is a same-origin request.

The background service worker schedules an alarm on the configured interval. When the alarm fires, it sends a message to any open Canvas tab. The content script receives the message, fetches the activity stream, compares item IDs against a list stored in `localStorage`, and shows the badge if anything is new.

Settings are stored in `chrome.storage.sync` so they follow the user across devices if they are signed into the browser.
